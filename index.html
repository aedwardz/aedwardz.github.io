Fullstack Search Engine utilizing Flask Framework

->GitHub Repository<-

Built a Fullstack search engine using Python for the backend and HTML/CSS for the frontend interface. It utilizes the Flask framework to establish communication between the backend and frontend, ensuring a seamless user experience. The backend functionality involves indexing documents stored in a zip file, extracting text content, and calculating TF-IDF (Term Frequency-Inverse Document Frequency) scores for each term. The engine merges partial indexes to accelerate search speed, achieving results in under 300 milliseconds. The ranking algorithm considers factors like page hits, TF-IDF scores, and the importance of text tags such as headers and bold tags to determine the relevance of documents. The frontend interface follows a minimalist design approach, prioritizing performance and result presentation. Overall, the project aims to deliver a robust and efficient search tool for users.

Explicit Dynamic Memory Allocator

->GitHub Repository<-

This dynamic memory allocator implements a custom memory allocation library with functions adjacent to malloc and free. The allocation function allocates uninitialized memory from the heap by inrementing the brk pointer. It ensures that the allocated memory is 16-byte aligned and returns a pointer to the allocated memory region. If the requested size is 0, it returns NULL and sets errno to ENOMEM. The memory deallocation function marks a dynamically allocated block as no longer in use, coalesces adjacent free blocks if necessary, and adds the block to the appropriate bucket according to the block placement policy. It checks the validity of the pointer being freed and sets errno accordingly in case of errors such as invalid pointers or mismatched block sizes. Additionally, it ensures that the allocated bit is properly set in the header and footer of the block. The code also includes helper functions for managing buckets, inserting and removing blocks, and handling memory coalescing.

Shell Interpreter in C

->GitHub Repository<-

The provided code is a C program that serves as a shell interpreter, allowing users to execute commands. It utilizes various signal handlers to handle events such as child process termination (`sigchld_handler`) and user-defined signals (`sigusr2_handler`). The program begins by initializing variables and setting up handlers for signals. It then enters a loop where it continuously prompts the user for input using the `readline` function. Within this loop, the program checks for various conditions such as background process termination (`sigchldFlag`), history commands, built-in commands like `cd` and `exit`, and multiple processes execution. For each command entered by the user, the program parses it, executes it, and waits for completion if it's a foreground job. Additionally, it maintains a history of commands entered by the user and handles input/output redirection and pipingfor processes. The program provides error handling for various scenarios and ensures proper memory management throughout its execution.
